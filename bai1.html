BÀI 1: THAO TÁC VỚI TỆP TIN VÀ THƯ MỤC

> Tổng quan Dự án:

  • Mục tiêu: Chuẩn hóa quy trình quản lý tài nguyên số trên môi trường Windows, khắc phục triệt để tình trạng phân mảnh dữ liệu học tập.
  • Giải pháp: Áp dụng tư duy phân cấp thư mục hình cây (Tree-structure) kết hợp chặt chẽ với bộ quy tắc đặt tên (Naming convention) tiêu chuẩn quốc tế.
  • Hiệu suất: Tối ưu hóa lên tới 40% thời gian truy xuất tài liệu so với phương pháp lưu trữ tự do.
  • Tác động dài hạn: Hạn chế tối đa các lỗi đường dẫn (path errors) khi đồng bộ hóa dữ liệu lên Cloud Storage (OneDrive, Google Drive), tạo tiền đề vững chắc cho các dự án làm việc nhóm (Teamwork).
▼ > Phần I: Xây dựng cấu trúc cây thư mục (Click để gập/mở)
  • Thiết lập thư mục gốc ThucHanh_PhanNhatQuan nằm an toàn trong vùng không gian làm việc Documents.
  • Phân cấp logic các lớp dữ liệu bên trong (Ví dụ: tạo thư mục con TaiLieu) để phân loại rạch ròi giữa file thực hành và file lý thuyết.
  • Cấu trúc này giúp quá trình sao lưu (backup) hoặc nén file (zip/rar) diễn ra nhanh chóng, không bị sót dữ liệu.
▼ > Phần II: Quy tắc đặt tên (Naming Convention)

Để đảm bảo hệ thống vận hành trơn tru và tương thích với mọi môi trường lập trình/máy chủ web, các quy tắc sau đã được áp dụng triệt để:

  • Quy tắc cốt lõi: Tuyệt đối không sử dụng tiếng Việt có dấu và khoảng trắng ở cả tên file lẫn tên thư mục gốc.
  • Đối với file Hệ thống/Code: Áp dụng chuẩn PascalCase (Viết hoa chữ cái đầu mỗi từ, ví dụ: GhiChuQuanTrong.txt, DiChuyen.txt) hoặc snake_case.
  • Đối với file Báo cáo/Tài liệu: Tuân thủ cấu trúc định danh [Mã_Môn]_[Tên_Bài]_[MSSV] (Ví dụ: INT1008_Bai1_25020769.pdf) để tối ưu việc nộp bài và phân loại.
  • Quản lý phiên bản (Versioning): Tránh việc lưu file kiểu "baocao_moi", "baocao_cuoi_cung". Thay vào đó, sử dụng hậu tố số hóa như _v1, _v2 và chốt bằng _Final.
